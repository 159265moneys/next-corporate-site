# 第1章: ゲームの正体（一行で）

**カオスな線画ランドマークが乱立する巨大空間迷路を、たった1本の正解ルートを探りながら
螺旋状に頂上まで登る、時間停止型・激ムズ空間アスレチック。**

確定式: `空間迷路 × 線画のみの幾何学建築 × 上に上がっていく × 解法は基本1パターン`
- シューティング要素は**なし**（過去に検討し廃止確定）
- ステージは**1個のみ**。Only Up要素少量（落下=進捗後退）
- **今回は白黒（モノクロ）でOK**。色は後日。ただし後述の反転演出で白黒のまま最強に見せる

## パレット（白黒版）
```
--paper:  #FFFFFF   背景（紙）
--ink:    #111111   線・文字（墨）
--faint:  #C9C9C9   補助線・偽ルートのヒント・方眼
```

# 第3章: ワールド構築仕様

## 3-1. 空間
- 一辺 **120u** の巨大立方体空間。床は方眼(faint色)の大地。頂上ゴールは高度 **約60u**（MVP）
- ランドマークは**統一性皆無に乱立**させる。道はぐっちゃぐちゃでいい。
  ただし全ランドマークは正解ルートの「道標」を兼ねる（攻略が「大仏の掌→鳥居の笠木」と語れる）

## 3-2. ランドマーク造形レシピ（全部LineSegments線画。寸法は目安）
1. **エッフェル塔** (h45u・空間の一角): 4本の脚を2次曲線プロファイル（基部オフセット6u→頂部収束）
   で立て、高度25%/55%に水平デッキ2枚、脚間をX字ラティスで接続。デッキは着地可能
2. **大仏** (h18u): 回転体(lathe)プロファイルで裾広がりの衣→胴→肩、球の頭部+肉髻、
   簡略化した印相の手（box合成）を膝上に。**掌は重要な足場**。頭部背後に光背リング（円線）
3. **鳥居** (h12u): 内傾4°の2本柱、上端に反り上がる笠木（曲線）、貫、中央に額束。**笠木は足場**
4. **観覧車** (r14u): リム円×2(z差)+スポーク16本+吊りゴンドラ12基(小box)+A字支持脚。
   **常時回転0.05rad/s。ゴンドラはキネマティック足場**（ルートの華。乗って高度を稼ぐ）
5. **五重塔** (h22u): 5段の縮小box積層+各層から張り出す庇スラブ+頂部相輪（縦線+小円環）。
   **庇が螺旋状の足場になる**
6. 余裕があれば: 公園（ブランコ・すべり台）、正体不明の巨大歯車機構、橋梁。なんでもあり
- 全ジオメトリは可能な限り**1つのLineSegmentsにマージ**して描画コストを抑える
  （回転する観覧車と可動ギミックだけ別オブジェクト）

## 3-3. レベルデータの実装戦略（一発で成立させる最重要方針）
**見た目(線画)と当たり判定を完全分離する。**
- コライダーは `{min:[x,y,z], max:[x,y,z], type}` の**AABB配列として手書きで作成**
  （ランドマークのメッシュから自動生成しない。制御不能になる）
- 正解ルートは `ROUTE = [ {pos, note} , ... ]` の**順序付きノード配列**として先に設計し、
  その線に沿って足場AABBを約40個、高度3層分、空間を螺旋周回するように配置する
- **偽ルート**: 本物そっくりの足場列を数本、途中で途切れる形で配置（探索の失敗も体験）
- 可動足場は毎フレームAABBを更新するキネマティック扱い。接地中のプレイヤーに足場速度を加算

# 第4章: プレイヤーと物理（数値は実測済みの推奨初期値）

## 4-1. 物理（固定タイムステップ厳守）
```
シミュレーション: dt=1/120 の accumulator 方式（描画は60fps）
重力 28 u/s² / ジャンプ初速 11.5 u/s（到達高さ≒2.3u）
地上加速 60 u/s² / 最高速 7.5 u/s / 地上摩擦 10/s / 空中制御係数 0.55
コヨーテタイム 0.09s / ジャンプ先行入力バッファ 0.1s
プレイヤー実体: 半径0.35uの球コライダー
```
- 衝突: 球vsAABBの離散解決（めり込み押し出し）+ 接地レイキャスト。物理エンジン不使用
- **コヨーテタイムとジャンプバッファは激ムズゲーの生命線。省略禁止**
- 落下: 下の構造物に着地=進捗後退として許容。`y < -5` のキルプレーンで飛散死→祠リスポーン

## 4-2. 粒子群体（プレイヤーの見た目=HP）
- THREE.Points 約600粒を半径0.45uの殻状に分布、各粒が中心へのlerp+ノイズドリフトで「群体」感
- **HP=粒子数**。危険ギミック掠り: -80粒+ノックバック、粒は物理的に飛散して消える
- 150粒未満で自動死亡。死=全粒飛散(0.3s)→祠へ**逆再生凝集**リスポーン(0.4s)。テンポ最優先
- 移動方向に群体がわずかに伸び、着地でつぶれる（squash&stretchを粒子分布で表現）

## 4-3. 時間停止（コアスキル）
```
持続 1.2s / クールタイム 6s / 発動キー Shift
```
- **worldTime と playerTime を分離**し、全ギミックは worldTime の純関数として駆動する
  （`angle = f(worldTime)`）。停止= worldTime の進行を止めるだけ。
  これで①全ギミックが決定論的=覚えゲーとして公平 ②停止実装が一箇所で完結
- 停止中もプレイヤー物理は進む（ジャンプ軌道ごと凍った刃の間をすり抜ける）
- リアルタイムでは**数学的に通過不可能な区間**を最低2箇所置く（唯一の解が「ここで止める」）
- 演出: 発動でネガ反転+無音化+落下粒子の空中静止、解除で軽いズームパンチと共に世界が戻る
- CT表示はプレイヤー周囲を回る円弧線リング（UIすら幾何学で）

# 第7章: 前任が踏んだ地雷（同じ穴に落ちるな）

1. **自動化タブ/プレビュータブは document.hidden=true でrAFが完全凍結する。**
   スクリーンショットが真っ白でも実環境のバグとは限らない。**動作の最終確認は必ず
   人間(発注者)に実ブラウザで依頼**し、自分はDOM/状態値の検証に徹すること
2. ヘッドレスプレビューは window.innerWidth が0を返すことがある。実Chrome基準で判断
3. rAF依存の初期化(ローディング演出等)には**必ずsetTimeoutのフェイルセーフを併設**
   （バックグラウンドタブで開かれたとき固まって操作不能になる）
4. innerHTMLでの要素ラップはclickリスナーを消す。クリック処理はdocumentへのイベント委譲で
5. `node --check js/*.js` を修正のたびに実行。コミット前に必ず全ファイル合格
6. requestAnimationFrameのdtは可変。**物理を可変dtで書くと再現性が死ぬ**(固定タイムステップ厳守)
