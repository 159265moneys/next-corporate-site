# 『NEXT FRAME』ゲームデザイン仕様書 v1.0
> 2026-07-07 設計セッションの確定事項。実装時はPLAYBOOK.mdの技術基準に従う。

## コンセプト（一行）
カオスな線画ランドマークが乱立する巨大立方体の空間迷路を、
たった1本の正解ルートを探りながら螺旋状に頂上まで登る、時間停止型・激ムズ空間アスレチック。

## 確定式
**空間迷路 × 線画のみの幾何学建築 × 上に上がっていく × 解法は基本1パターン**
- シューティング: **廃止**（確定）
- ステージ: **1個のみ**。巨大立方体空間。最終ゴールは頂上

## ワールド
- 巨大立方体の空間に、高さまばらな巨大構造物が**統一性皆無**に乱立する
- ランドマーク例（全部ワイヤーフレーム線画のジオメトリで様式化）:
  エッフェル塔 / 大仏 / 神社(鳥居・本殿) / 公園(遊具・観覧車) / 五重塔 / 橋梁 /
  クレーン / 螺旋塔 / 巨大歯車機構 / 何に使うか不明なオブジェ
  — 「幾何学で立体にして見栄えがいいもの」なら何でもあり。道はぐっちゃぐちゃでいい
- **ランドマークは攻略言語**: 統一性のなさは欠陥ではなく機能。
  「大仏の掌→鳥居の笠木」のようにルートが固有名詞で記憶・共有される
- 描画: 建築は線画のみ(LineSegments、墨色 #1A2333)。通常時は線上を微かな光パルスが流れる。
  マージ済みBufferGeometryで一括描画しパフォーマンス確保

## 進行構造
- 正解ルートは**1パターンのみ**。全体としては空間を螺旋状に周回しながら上昇し、頂上でクリア
- 偽ルート・見せかけの足場を大量に配置（探索の失敗も体験の一部）
- **高度＝進捗**。画面端に現在高度メーター(線画)を常時表示
- Only Up要素(少量): **落下は基本即死ではなく「下の構造物に落ちて進捗後退」**。
  ただし危険ギミック(刃・クラッシャー・レーザー)は粒子削り〜即死
- チェックポイントは疎らに配置(「祠」の線画オブジェ)。祠間は覚えゲーとして歯応えを出す

## プレイヤー
- 青い粒子の群体(accent #2563EB)。線画の世界で唯一の「粒子でできた存在」
- HP＝粒子数。掠りダメージで粒子が物理的に飛散して体が薄くなる。全損で飛散死→祠から逆再生凝集リスポーン(0.5秒以内)
- 死亡回数カウンター常時表示

## 時間停止（コアスキル）
- 発動1.2秒完全凍結 / CT6秒。自分の慣性は生きる(ジャンプ軌道ごとすり抜ける)
- リアルタイムでは数学的に通過不可能な区間を意図的に置き、「どこで止めるか」の配分がルート構築になる
- 演出: 停止中は線のパルスが静止し全建築が青転調、粒子(滝・雨)が空中静止、音が消え、
  自分だけが動く。解除でFOVパンチと共に世界が帰ってくる
- CT表示は自分の周囲を回る線のリング

## ギミック語彙（全部ジオメトリ・粒子・線）
掃引レーザー格子 / 旋回サーチライト光錐 / スラム式クラッシャー / 回転ワイヤーブレード /
触れると粒子化して崩落する足場 / 粒子の滝・奔流(押し流し) / 振り子刃 / 時限シャッター /
回転する構造物自体が足場(観覧車・歯車) / 重力反転ゾーン(粒子が逆さに降って事前告知)

## カメラ・操作
- アイソメ寄りのOrthographic追従。停止中は微ドリー+ロールで演出
- PC特化: WASD移動 + Space跳躍 + Shift時間停止。モバイルは対象外(精度が出ない)

## MVPスコープ
1. 立方体空間+ランドマーク5基(エッフェル塔・大仏・鳥居・観覧車・五重塔)の線画生成
2. 粒子群体プレイヤー(移動・ジャンプ・削り・飛散死・凝集リスポーン)
3. 時間停止(演出込み) + 停止必須ギミック2箇所
4. 正解ルート1本(高度3層分・祠1箇所) + 偽ルート数本
5. 高度メーター・死亡カウンター・クリア演出(全粒子が頂上で「NEXT FRAME」に収束)
